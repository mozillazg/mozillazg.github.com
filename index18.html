[django]修复 "TypeError: delete() got an unexpected keyword argument 'using'"

Date 周日 13 十月 2013 Category django Tags python / django

调用 delete 方法时，下面的代码会导致出现标题中的错误：

Foo.objects.filter(name='foo').delete(using='writedb')

原因是因为只有单个查询结果对象的 delete 方法拥有 using 参数， 而查询结果集对象的 delete 方法没有 using 关键字参数:

foos = Foo.objects.filter(name='foo')
for foo in foos:
    foos.delete(using='writedb')

[django]修复 "dictionary update sequence element 0 has length 1, 2 is required"

Date 周日 13 十月 2013 Category django Tags python / django

定义 url 路由时，下面的代码会导致出现标题中的错误：

url(r'^foo/$', 'view_func_name', 'foo')

原因是因为忘了写 name 这个关键字参数:

url(r'^foo/$', 'view_func_name', name='foo')

[python]for 循环中的局部变量陷阱

Date 周日 13 十月 2013 Category python Tags python

先看一段代码：

>>> x = 10
>>> [x for x in range(3)]
[0, 1, 2]
>>> x
2

从这段代码，我们可以知道：for 循环中用于循环主体的变量会影响上下文的局部变量。
