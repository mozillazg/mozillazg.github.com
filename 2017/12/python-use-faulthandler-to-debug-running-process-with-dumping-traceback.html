Python: 使用 faulthandler 模块获取运行中进程的 traceback 信息

2017-12-09

python Tags debug / faulthandler

Python 3.3 新增了一个 faulthandler 模块，可以输出运行中进程的 traceback 信息，可以用来作为调试运行中的 Python 进程的一种手段。

特点及限制¶

模块本身是使用 C 实现的，所以有如下特点和限制：

特点¶

  • 在 Segmentation fault 或者 Python 死锁时仍然可以输出 traceback 信息

限制¶

  • 只能用于信号安全的函数
  • 只支持 ASCII 字符。程序使用 backslashreplace 作为编码时的 error handler
  • 每个字符串最长只能包含 500 个字符
  • 只显示文件名，函数名和行号（不显示源码）
  • 最大支持 100 个 frame 和 100 个线程
  • 显示顺序是翻转的：最近的调用显示在前面

安装¶

Python 3.3 自带了这个模块，Python 2.x 可以通过一下方式安装:

pip install faulthandler

使用示例¶

手动输出 traceback 信息¶

测试函数:

from threading import Thread
import time


def do(x):
    x = x * 3
    time.sleep(x * 60)


def main():
    threads = []
    for x in range(1, 3):
        t = Thread(target=do, args=(x,))
        t.start()
    for x in threads:
        x.join()

通过信号自动输出 traceback 信息¶

给前面的测试函数增加信号处理，增加通过 USR1 信号输出 traceback 信息的功能：

import faulthandler
import signal
from threading import Thread
import time


def do(x):
    x = x * 3
    time.sleep(x * 60)


def main():
    threads = []
    for x in range(1, 3):
        t = Thread(target=do, args=(x,))
        t.start()
    for x in threads:
        x.join()


if __name__ == '__main__':
    faulthandler.register(signal.SIGUSR1)
    main()

运行程序，然后就可以通过向进程发送 USR1 信号来获取运行中进程的 traceback 信息了:
