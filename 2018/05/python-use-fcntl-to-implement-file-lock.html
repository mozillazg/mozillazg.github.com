Python: 使用 fcntl 实现文件锁

2018-05-12

python Tags fcntl / proc / inotify

前言¶

文件锁是一种在特定时间只允许一个用户或进程拥有对特定计算机文件的访问权限的机制。计算机系统实现该特性以避免竞争读写。[1]

文件锁的常用用途如下：

  • 保证某个服务只有一个进程在运行，比如通过 pid 文件实现一个服务只能启动一次，再次启动会提示服务已运行。[2] [3] [4]
  • 实现不同进程之间的互斥锁，同一时刻只允许一个进程做某个操作。比如有多个进程都去操作某个文件，但是只允许一个进程进行写入操作，其他进程只能进程读取操作。通过文件锁实现简单的进程间的 master/slave 或 leader/follower 模式。 [5] [6] [7]

使用 fcntl 模块使用文件锁

在 linux 下可以使用 fcntl 这个模块来实现文件锁的功能。

监听文件锁变更事件¶

实际上并没有一个文件变更事件直接对应文件锁，不过我们可以通过 IN_CLOSE 事件来监听文件锁变化，因为如果文件 fd 被关闭了的话，文件锁肯定就被释放了。
