让我们一起来阅读源代码之 Howdoi （二）

2018-04-22

python Tags let-us / let-us-read-source-code / Howdoi

前言¶

在上篇文章中，我们熟悉了 howdoi 这个项目的基本功能，下面我们正式进入代码相关阶段。

程序入口¶

要阅读源代码首先需要找到程序的入口，通过入口我们就可以大致了解程序的主流程。

因为 howdoi 这个项目是一个命令行工具，根据经验 [1] [2] ，可以通过项目的 setup.py 文件找到命令行程序的入口。一般命令行程序的项目下 setup.py 中 entry_points 中定义了程序的入口，howdoi 这个项目的 setup.py 中 entry_points 的 内容如下 :

entry_points={
    'console_scripts': [
        'howdoi = howdoi.howdoi:command_line_runner',
    ]
},

其中 console_scripts 的值定义了各个命令行工具的入口，这里表示 howdoi 这个命令行工具的入口是 howdoi.howdoi:command_line_runner ，即 howdoi.howdoi 模块中的 command_line_runner 函数。我们可以验证一下:

$ python -c 'import howdoi.howdoi; howdoi.howdoi.command_line_runner()'
usage: -c [-h] [-p POS] [-a] [-l] [-c] [-n NUM_ANSWERS] [-C] [-v]
          [QUERY [QUERY ...]]

instant coding answers via the command line

positional arguments:
  QUERY                 the question to answer

optional arguments:
  -h, --help            show this help message and exit
  -p POS, --pos POS     select answer in specified position (default: 1)

                        number of answers to return
  -C, --clear-cache     clear the cache
  -v, --version         displays the current version of howdoi

可以看到 howdoi.howdoi.command_line_runner 确实就是命令行的入口。

command_line_runner 初探¶

既然找到了入口，我们先来看一下入口函数 command_line_runner ， command_line_runner 函数的内容如下:

def command_line_runner():
    parser = get_parser()
    args = vars(parser.parse_args())


    if args['version']:
        print(__version__)
        return


    if args['clear_cache']:
        _clear_cache()
        print('Cache cleared successfully')
        return


    if not args['query']:
        parser.print_help()
        return


    # enable the cache if user doesn't want it to be disabled
    if not os.getenv('HOWDOI_DISABLE_CACHE'):
        _enable_cache()


    if os.getenv('HOWDOI_COLORIZE'):
        args['color'] = True


    utf8_result = howdoi(args).encode('utf-8', 'ignore')
    if sys.version < '3':
        print(utf8_result)
    else:
        # Write UTF-8 to stdout: https://stackoverflow.com/a/3603160
        sys.stdout.buffer.write(utf8_result)
    # close the session to release connection
    howdoi_session.close()

我们可以通过增加一些 print 语句的方式来看看判断函数中各部分都实现了啥功能:

def command_line_runner():
    parser = get_parser()
    args = vars(parser.parse_args())
    print('== args ==: {}\n'.format(args))

    # 省略未修改的部分...

然后我们执行一下 howdoi 命令，看看结果:

$ export HOWDOI_SEARCH_ENGINE=bing   # 别忘了这一步，因为默认用的搜索引擎是 google
$ howdoi python convert int to str

通过输出我们可以发现：args 中包含了我们再命令行中输入的参数 python convert int to str 即我们要搜索的问题，utf8_result 就是命令行的结果。所以，我们可以判断出：解析命令行参数的工作由

parser = get_parser()
args = vars(parser.parse_args())

实现，搜索问题并得到结果由

utf8_result = howdoi(args).encode('utf-8', 'ignore')

我们先来看核心的 howdoi(args) 部分。

howdoi(args)¶

howdoi 函数的代码如下：

def howdoi(args):
    args['query'] = ' '.join(args['query']).replace('?', '')
    try:
        return _get_instructions(args) or 'Sorry, couldn\'t find any help with that topic\n'
    except (ConnectionError, SSLError):
return 'Failed to establish network connection\n'

通过代码我们可以发现 howdoi 的主要功能是由 _get_instructions 实现的。 这样的话，我们来看一下 _get_instructions 函数。

_get_instructions¶

_get_instructions 函数的代码如下：

def _get_instructions(args):
    links = _get_links(args['query'])
    if not links:
        return False

    question_links = _get_questions(links)
    if not question_links:
        return False

    only_hyperlinks = args.get('link')
    star_headers = (args['num_answers'] > 1 or args['all'])

    answers = []
    initial_position = args['pos']
    spliter_length = 80
    answer_spliter = '\n' + '=' * spliter_length + '\n\n'

    for answer_number in range(args['num_answers']):
        current_position = answer_number + initial_position
        args['pos'] = current_position
        link = get_link_at_pos(question_links, current_position)
        answer = _get_answer(args, question_links)
        if not answer:
            continue
        if not only_hyperlinks:
            answer = format_answer(link, answer, star_headers)
        answer += '\n'
        answers.append(answer)
return answer_spliter.join(answers)

看起来实现的功能比较多，先也用 print 来看看内部实现的各种功能：

通过上面的结果，我们可以初步得出如下结论：

  • _get_links 实现了搜索问题获取 stackoverflow 网页地址的功能
  • _get_answer 实现了从某个 stackoverflow 网页中获取答案的功能

可以看出来 _get_instructions 中比较关键的是 _get_links 和 _get_answer ， 也可以说是 howdoi 这个程序中比较关键的两个函数。关于这两个函数我们下次再看。

总结¶

本文我们正式开始阅读 howdoi 项目的源代码了，通过阅读源代码，我们知道了如下信息：

  • 通过 setup.py 文件我们得知了程序的入口是 howdoi.howdoi.command_line_runner 函数。
  • 通过分析 command_line_runner 得知 _get_instructions 实现的主要的功能
  • 而通过分析 _get_instructions 函数我们得知 _get_links 和 _get_answer 函数是关键的两个函数。

下次我们将分析 _get_links 和 _get_answer 这两个函数。
